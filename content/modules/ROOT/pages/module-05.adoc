= Configure Integrity Controls

== Overview

To mitigate situations where a trusted binary is substituted, fapolicyd supports different types of file integrity:

* **File-size checking** - A very fast method where fapolicyd verifies that the file is the correct size before allowing execution
* **SHA-256 hashes** - Computing and checking SHA-256 checksums is more secure than file-size checking, but does have an impact on system performance
* **Integrity Measurement Architecture (IMA)** - The IMA subsystem can be used by fapolicyd to support file integrity checks

For this scenario, we're going to use SHA-256 hashes to verify file integrity. This provides a good balance between speed and security, and is simple to set up and maintain.

== Step 1: Enable SHA-256 Integrity Checking

Configure integrity in the `/etc/fapolicyd/fapolicyd.conf` file:

[source,bash,role=execute]
----
sudo sed -i 's/integrity.*/integrity = sha256/' /etc/fapolicyd/fapolicyd.conf
----

== Step 2: Restart the Daemon

Restart the daemon to apply the configuration change:

[source,bash,role=execute]
----
sudo systemctl restart fapolicyd
----

== Step 3: Verify Integrity Checking

We can now verify whether file integrity checking is performed:

[source,bash,role=execute]
----
/bin/more "mooooooo"
----

[source,text]
----
-bash: /bin/more: Operation not permitted
----

Success! We've now mitigated the risk of an attacker hijacking a trusted binary on the system by configuring fapolicyd integrity controls.

== Summary

In this module, you've learned:

* The three integrity checking methods supported by fapolicyd: file-size, SHA-256, and IMA
* How to enable SHA-256 integrity checking in `/etc/fapolicyd/fapolicyd.conf`
* How integrity checking blocks execution when a trusted binary has been substituted
