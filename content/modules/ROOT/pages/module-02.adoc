## Step 2: Verify fapolicyd Status


= Installation & Configuration

== Overview

The File Access Policy Daemon (fapolicyd) controls the execution of applications based on a user-defined policy. It is one of the most effective ways to mitigate the risk of untrusted and possibly malicious applications on the system.

Application control helps protect against unauthorized process execution. When implemented correctly, fapolicyd can block binaries that are not trusted by the system's configured trust sources.

[NOTE]
====
This lab uses RHEL, where the fapolicyd package is available but the service is disabled by default until you explicitly enable it.
====

== Step 1: Verify fapolicyd Installation

First, verify that fapolicyd is installed on your Red Hat Enterprise Linux system:

[source,bash,role="execute",subs=attributes+]
----
sudo systemctl status fapolicyd
----

The service exists but is disabled by default on installation:

[source,text]
----
○ fapolicyd.service - File Access Policy Daemon
     Loaded: loaded (/usr/lib/systemd/system/fapolicyd.service; disabled; preset: disabled)
     Active: inactive (dead)
----

== Step 2: Review Default Configuration

Next, inspect the fapolicyd configuration that has been created:

[source,bash,role="execute",subs=attributes+]
----
sudo cat /etc/fapolicyd/fapolicyd.conf
----

Several important settings are configured by default:

* **permissive = 0** - The daemon runs in enforcing mode, meaning all unauthorized processes will be blocked. You can optionally configure permissive mode to log unauthorized processes rather than block them.
* **uid = fapolicyd** - A dedicated user has been created on the system for the daemon.
* **Trust sources** - The default sources of trust are the RPM database and a file-backed trust database. Any packages installed into the RPM database are trusted by default. The file at `/etc/fapolicyd/fapolicyd.trust` can be used for third-party binaries and other applications not present in the RPM database.
* **integrity = none** - There is currently no integrity checking configured.

[TIP]
====
The file-backed trust database at `/etc/fapolicyd/fapolicyd.trust` is useful for allowing third-party or locally compiled binaries without packaging them as RPMs.
====
