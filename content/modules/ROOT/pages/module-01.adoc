## Step 1: About This Lab

== Objective

By the end of this guided lab exercise, you will be able to:

* Deploy and manage application control on Red Hat Enterprise Linux systems.
* Install the File Access Policy Daemon (fapolicyd).
* Verify fapolicyd configuration and block unauthorized applications.
* Create a file-based trust database to allow trusted applications.
* Configure integrity checking for fapolicyd.

== Introduction

Learn how to configure and use file access policy in Red Hat Enterprise Linux.

In this lab, you will configure application control for Red Hat Enterprise Linux systems. Application control is a security approach designed to protect against unauthorized process execution on systems. When implemented correctly, application control helps mitigate the risk of malicious code (malware), such as ransomware, executing on a system.

The File Access Policy Daemon (fapolicyd) controls the execution of applications based on a user-defined policy, and is one of the most effective ways to mitigate the risk of untrusted and possibly malicious applications on the system.

=== Lab Modules

. *Update* — Update the trust database to allow trusted applications
. *Explore* — Explore fapolicyd integrity checking capabilities
. *Integrity Controls* — Configure SHA-256 integrity checking
