= Security: File Access Policy (fapolicyd) Lab

== Overview

Learn how to configure and use file access policy in Red Hat Enterprise Linux.

*Maintainer:* Nate Lager (nlager@redhat.com)

== Description

After completing this scenario, users will be able to deploy and manage application control on Red Hat Enterprise Linux systems.

== Concepts Covered

* Install File Access Policy Daemon (fapolicyd)
* Verify fapolicyd configuration and block unauthorized applications
* Create a file-based trust database to allow trusted applications
* Configure integrity checking for fapolicyd

== Use Case

In this lab, you are going to configure application control for Red Hat Enterprise Linux systems. Application control is a security approach designed to protect against unauthorized process execution on systems. When implemented correctly, application control helps mitigate the risk of malicious code (malware), such as ransomware, executing on a system.

== Time to Complete

Approximately 30 minutes

== Lab Structure

This lab contains 5 modules:

1. *Install* - Install and configure the File Access Policy Daemon
2. *Verify* - Verify fapolicyd is blocking unauthorized applications
3. *Update* - Update the trust database to allow trusted applications
4. *Explore* - Explore fapolicyd integrity checking capabilities
5. *Integrity Controls* - Configure SHA-256 integrity checking
